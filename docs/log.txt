    PUT-annotations route now calls jobs.schedule(slide_id) after saving; the worker
    coalesces rapid edits and runs extract_patches -> embed_slide -> train (global).
    Graceful states: need_2_classes/no_data -> "waiting", EmbedderError -> "error",
    success -> "ready". Exposed via GET /api/learn/status.

    and a live similarity request don't hit torch/GPU concurrently.
  * Similarity is now annotation-based: models.SimilarityRegion carries the full W3C

    MEAN embedding of the annotation's tissue tiles (_annotation_reference_vector,

    empty_annotation -> 400.
  * Frontend: removed the Extract/Train top-bar buttons and the patch-montage +

2026-07-08 — Predict view: region boundaries instead of a heatmap.
  "Predict view" now outlines the regions the trained head calls the selected
  class with >= 80% probability, rather than painting a blue->red heatmap. Outline
  only, tile-grid (blocky) edges, fixed 80% threshold. "Find similar" (annotation
  similarity) is unchanged and still uses the heatmap.
  * backend/inference.py: added PREDICT_MIN_CONFIDENCE = 0.8; predict() now returns
    boundary SEGMENTS ([[x1,y1,x2,y2],...] in region-local px, via new
    _boundary_segments: an inside tile (score>=thr) emits each edge bordering a
    non-inside neighbor) plus n_above + min_confidence, and no longer writes a
